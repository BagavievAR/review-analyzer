# Web App for Review Analysis

# Багавиев Артем 6310

Учебный проект: веб-приложение на Python/Flask для сбора и анализа пользовательских отзывов с использованием методов обработки естественного языка.

## Tech Stack

- Python
- Flask
- spaCy
- SQLite
- Bootstrap 5
- Docker

## Features

- Сбор пользовательских отзывов через веб-интерфейс
- Хранение отзывов в базе данных (SQLite)
- Анализ тональности и ключевых слов с помощью spaCy
- Фильтрация отзывов по тональности, оценке и ключевому слову
- Улучшенный UI на Bootstrap 5 с карточками отзывов и цветовой индикацией тональности
- Контейнеризация приложения с использованием Docker и Docker Compose

## How to Run (local, via venv)

1. Клонировать репозиторий:
   ```bash
   git clone https://github.com/BagavievAR/review-analyzer.git
   cd review-analyzer
   ```

2. Создать и активировать виртуальное окружение:
   ```bash
   python -m venv venv

   # Windows (PowerShell)
   .\venv\Scripts\Activate.ps1

   # или Windows (cmd)
   venv\Scripts\activate.bat
   ```

3. Установить зависимости:
   ```bash
   pip install -r requirements.txt
   ```

4. Инициализировать базу данных:
   ```bash
   flask --app run init-db
   ```

5. Запустить приложение:
   ```bash
   python run.py
   ```

6. Открыть в браузере:
   - http://127.0.0.1:5000/

## How to Run (Docker)

1. Клонировать репозиторий:
   ```bash
   git clone https://github.com/BagavievAR/review-analyzer.git
   cd review-analyzer
   ```

2. Убедиться, что установлены Docker и Docker Compose.
   Проверить можно такими командами:
   ```bash
   docker --version
   docker compose version
   ```

3. В корне проекта выполнить сборку и запуск контейнера:
   ```bash
   docker compose up --build
   ```

4. В отдельном терминале инициализировать базу данных внутри контейнера:
   ```bash
   docker compose exec web flask --app run init-db
   ```

5. Открыть приложение в браузере:
   - http://127.0.0.1:5000/

6. Остановить контейнеры:
   ```bash
   docker compose down
   ```

7. Для повторного запуска без пересборки:
   ```bash
   docker compose up
   ```

Приложение настроено так, чтобы SQLite-база данных сохранялась в отдельной папке `data/`, подключаемой в контейнер через Docker Compose.
